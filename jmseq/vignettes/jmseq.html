The computation in this vignette is performed by the Python package jmseq, called from R via the reticulate package. Results are returned to R as ordinary data frames and lists, and displayed using ggplot2 and kableExtra.

The jmseq Python package fits joint models of longitudinal biomarker trajectories and time-to-event outcomes. The longitudinal submodel is a continuous-time linear Gaussian state space model (LGSSM) estimated by MAP inference using a JAX Kalman filter and NumPyro. The survival submodel is a Poisson hazard model that takes the Kalman-filtered states as time-varying covariates.

This vignette demonstrates 4-fold cross-validated prediction of mortality on the Mayo Clinic primary biliary cirrhosis (PBC) dataset, comparing a linear mixed model (random slopes only) to a fuller SDE model with autoregressive drift and diffusion.

Running Python from R with reticulate

The reticulate package embeds a Python session inside the R session. Three functions are used throughout this vignette:

  • import("module") — imports a Python module and returns an R object whose $ operator maps to Python attribute access. For example, jm <- import("jmseq") makes every public function in jmseq available as jm$function_name(...).
  • r_to_py(x) — converts an R object to its Python equivalent. R data frames become pandas.DataFrame objects; R vectors become numpy arrays.
  • py_to_r(x) — converts a Python object back to R. pandas.DataFrame → R data frame; Python lists and dicts → R lists.

Python objects returned by functions called via $ are converted to R automatically when they are standard types (data frames, numeric scalars, strings, lists). No explicit py_to_r() call is needed in most cases.

Dataset

The PBC dataset comprises 1945 longitudinal measurements of two biomarkers (log-bilirubin and albumin, both standardised) on 312 individuals followed for mortality up to 15 years from baseline. Time-invariant covariates are sex, age at baseline, and treatment assignment.

The data are loaded from the R package data object pbc, then converted to plain R data frames before being passed to Python.

## Path relative to the vignette directory (jmseq/vignettes/ → jmseq/data/)
load("../data/pbc.rda")

## Convert the treatment factor to a binary integer (D-penicillamine = 1).
## The Python pipeline works with numeric covariates only.
dataSurv_r <- as.data.frame(pbc$dataSurv)
dataSurv_r$trt_binary <- as.integer(dataSurv_r$trt == "D-penicil")
dataSurv_r <- dataSurv_r[, c("id", "sex", "agebaseline", "trt_binary",
                              "Time.cens", "event")]

dataLong_r <- as.data.frame(pbc$dataLong[, c("id", "Time",
                                              "logBili", "albumin")])

biomarkers     <- c("logBili", "albumin")
timeinvar_surv <- c("sex", "agebaseline", "trt_binary")

cat(sprintf("dataSurv: %d individuals, %d events\n",
            nrow(dataSurv_r), sum(dataSurv_r$event)))
#> dataSurv: 312 individuals, 140 events
cat(sprintf("dataLong: %d observations\n", nrow(dataLong_r)))
#> dataLong: 1945 observations

Preprocessing: interval splitting

Follow-up time is split into intervals of at most 0.25 years so that the Poisson hazard model approximates a continuous-time process. Intervals containing a biomarker observation are flagged so the Kalman filter performs an update step; empty intervals carry only a prediction step.

The R data frames are passed to jm$split_surv_long() using r_to_py(). The function returns a Python pandas.DataFrame, which reticulate automatically converts to an R data frame when it is assigned to an R variable.

## r_to_py() converts R data.frames to pandas DataFrames.
surv_py <- r_to_py(dataSurv_r)
long_py <- r_to_py(dataLong_r)

## jm$split_surv_long() is the Python function jmseq.split_surv_long().
## The returned pandas DataFrame is automatically coerced to an R data frame.
split_df <- jm$split_surv_long(
  surv_py,
  long_py,
  max_interval   = 0.25,
  timeinvar_surv = timeinvar_surv,
  biomarkers     = biomarkers
)

dataSurv_split <- split_df[c("id", "Time", "tstop", "event", timeinvar_surv)]
dataLong_split <- split_df[c("id", "Time", "tstop", biomarkers)]

cat(sprintf("After splitting: %d rows, max T per individual = %d\n",
            nrow(split_df),
            max(table(py_to_r(split_df["id"])))))
#> After splitting: 8993 rows, max T per individual = 64

Model variants

We compare two LGSSM variants for the biomarker trajectories. The five available variants are pre-defined in jmseq.VARIANTS; make_config() selects one by name and sets the state dimension p. Calling make_config() via reticulate returns a Python LGSSMConfig dataclass, which reticulate exposes as an R environment whose fields are accessed with $.

## make_config() returns a Python LGSSMConfig dataclass.
## Reticulate exposes Python dataclasses as R environments; fields are accessed with $.
## Note the 2L — R passes plain 2 as a double (float), not an integer, so Python
## keyword arguments that expect int must be annotated with L.
lmm_cfg  <- jm$make_config("model_lmm",          p = 2L)
full_cfg  <- jm$make_config("model_lmmdriftdiff", p = 2L)
cat("model_lmm:          free_drift =", lmm_cfg$free_drift,
    " free_diffusion =", lmm_cfg$free_diffusion, "\n")
#> model_lmm:          free_drift = FALSE  free_diffusion = FALSE
cat("model_lmmdriftdiff: free_drift =", full_cfg$free_drift,
    " free_diffusion =", full_cfg$free_diffusion, "\n")
#> model_lmmdriftdiff: free_drift = TRUE  free_diffusion = TRUE

The two models differ in whether the SDE drift matrix A and diffusion matrix G are estimated:

Model Drift A Diffusion G Random slope b Continuous-time SDE
model_lmm fixed 0 fixed 0 free Random slopes only; reduces to a linear mixed model
model_lmmdriftdiff free free free Full SDE with mean reversion and stochastic volatility

Cross-validated prediction

4-fold cross-validation is performed with a landmark time of 5 years; events are predicted over the window 5–15 years. For each fold:

  1. The LGSSM is fitted by MAP to the training individuals’ longitudinal data.
  2. The Kalman filter is run over all individuals. For test-fold individuals, biomarker observations after the landmark time are withheld; the filter extrapolates the last known state using the SDE dynamics alone.
  3. A Poisson GLM is fitted to the training data using the Kalman-filtered states as time-varying covariates.
  4. Event probabilities are predicted for test-fold individuals over post-landmark intervals.

jm$cross_validate() orchestrates all four steps. It accepts R data frames directly; reticulate converts them to pandas.DataFrame automatically before the call.

Model variants are specified via model_names — a plain R character vector of model name strings. Passing model names rather than Python config objects avoids the complexity of marshalling Python dataclass instances across the R/Python boundary: the Python function constructs the LGSSMConfig objects itself from the names.

cross_validate() returns a Python dict mapping model name to a CVResult object. Reticulate converts this dict to an R named list, and the CVResult fields are accessible with $.

## cross_validate() runs the full CV loop in Python.
## model_names is passed as a plain R character vector — no Python objects cross
## the language boundary.  R data frames are accepted directly and converted
## to pandas DataFrames automatically by reticulate.
## The returned dict of CVResult objects becomes an R named list;
## cv_results[["model_lmm"]]$pooled is an R list of performance statistics.
cv_results <- jm$cross_validate(
  dataSurv_split,
  dataLong_split,
  biomarker_cols = biomarkers,
  timeinvar_surv = timeinvar_surv,
  model_names    = c("model_lmm", "model_lmmdriftdiff"),
  p              = 2L,
  landmark_time  = 5.0,
  n_folds        = 4L,
  random_seed    = 1234L,
  lgssm_steps    = 3000L,
  poisson_steps  = 3000L,
  verbose        = TRUE
)

Results

Kalman filter trajectories

The Kalman filter output is stored in the per-fold prediction data frames inside cv_results. Each data frame contains the Kalman-filtered biomarker state means alongside the event and interval timing columns. py_to_r() converts the Python data frame to an R data frame for plotting with ggplot2.

The plot shows the raw biomarker observations (points) and the Kalman-filtered state means (lines) for five individuals from the first cross-validation fold, using model_lmmdriftdiff. Trajectories continue beyond the last observation date because the filter extrapolates using the fitted SDE.

pred1    <- py_to_r(cv_results[["model_lmmdriftdiff"]]$fold_pred[[1]])
pred1_dt <- as.data.table(pred1)

Per-fold predictive performance

cv_results[["model_lmm"]]$fold_stats is a Python list of dicts, which reticulate converts to an R list of named lists. Standard R functions (mapply, sapply, etc.) work directly on this structure.

get_fold_table <- function(model_name) {
  stats_list <- cv_results[[model_name]]$fold_stats
  rows <- mapply(function(s, i) {
    data.frame(
      Fold          = i,
      Observed      = s[["Observed"]],
      Predicted     = round(s[["Predicted"]], 1),
      `Log score`   = round(s[["Log score"]], 2),
      `C-statistic` = round(s[["C-statistic"]], 3),
      check.names   = FALSE
    )
  }, stats_list, seq_along(stats_list), SIMPLIFY = FALSE)
  do.call(rbind, rows)
}

The C-statistic and log-score are both computed over all person-time intervals from the landmark time to exit, treating each interval as an independent binary observation (event / no event). The C-statistic is the area under the ROC curve of the predicted interval-event probability; the log-score is the sum of the Bernoulli log-likelihoods.

Methods summary

The two-step procedure is:

  1. LGSSM — a continuous-time SDE \[dX(t) = (AX(t) + b)\,dt + G\,dW(t), \quad y_t = x_t + v_t, \quad v_t \sim N(0,R)\] is fitted to the longitudinal data by MAP inference (NumPyro SVI with an AutoDelta guide and ClippedAdam optimiser). Discrete-time transition matrices \(A_d(\Delta t) = \exp(A\,\Delta t)\) and process noise \(Q_d(\Delta t)\) are obtained via the Van Loan method, implemented with a fixed-order degree-7 Padé matrix exponential that contains no data-dependent conditionals and therefore compiles efficiently under JAX.

  2. Kalman filter — implemented in JAX using jax.lax.scan over time steps and jax.vmap over individuals. Filtered state means \(\hat x_t\) are extracted for each person-time interval. In the test fold, biomarker observations after the landmark time are withheld; the filter propagates the last observed state forward using only the SDE dynamics.

  3. Poisson GLM — the log-hazard is modelled as \[\log \lambda_i(t) = \beta_0 + \beta_t t + \beta_x \hat x_i(t) + \beta_z z_i\] where \(z_i\) are time-invariant covariates. Fitted by MAP using QR decomposition of the design matrix for numerical stability.
