R package jmseq: fitting joint models of longitudinal and time to event data by sequential Bayesian updating

Paul McKeigue

Joint modelling of longitudinal and time to event data can be efficiently implemented by sequential Bayesian updating. I describe the theory of this approach in a paper now under peer review: "Fitting joint models of longitudinal observations and time to event by sequential Bayesian updating".

This vignette demonstrates an approximate two-step procedure, in which a linear Gaussian state space model is fitted to the longitudinal data, latent states at the start of each person-time interval are sampled from the forward updates generated by the Kalman filter, and these latent state values are plugged into a Poisson regression model for the event status at the end of each person-time interval. Work is in progress to develop a one-step procedure. The R code in this tutorial can be used as a template for analysis of any dataset with longitudinal measurements of biomarkers (transformed to have an approximately gaussian distribution) and time to event.

Dataset

Next we load an example dataset: the Mayo Clinic primary biliary cirrhosis dataset. This comprises 1945 longitudinal observations of two biomarkers -- bilirubin and albumin -- on 312 individuals followed for mortality up to 15 years from baseline. There are two time-invariant covariates: sex and age at baseline, with treatment as an extra time-invariant covariate included in the survival dataset. During follow-up there were 140 deaths. Of these 48 deaths occurred during the 623.2 person-years of follow-up after the first 5 years.

The data are provided in two tables. dataLong contains the biomarker data, with one row per person per date of observation. The character vector biomarkers specifies the names of the biomarker variables, and timeinvar.long specifies the names of the time-invariant covariates in the longitudinal submodel. dataSurv contains the time to censoring with an indicator for event status (0 or 1) at the time of censoring, and timeinvar.surv specifies the names of the time-invariant covariates in the event submodel.

For joint modelling, we reformat the dataset as person-time intervals. Person-time intervals longer than max.interval are split using the function split.SurvLong. We set max.interval to 0.25 years as a compromise between accuracy and computation time. The new intervals have missing values for the biomarker observations, and event status set to 0.

data(pbc, package="jmseq")
list2env(pbc, envir=environment())
#> <environment: R_GlobalEnv>

nfolds <- 5
landmark.time <- 5
maxtime <- 15
set.seed <- 2345
max.interval <- 0.25

library(survival)

surv.split <- split.SurvLong(pbc$dataSurv, pbc$dataLong, max.interval=max.interval)
keep.long <- c("id", "Time", "tstop", timeinvar.long, biomarkers)
keep.surv <- c("id", timeinvar.surv, "Time", "tstop", "event") 
dataLong <- surv.split[, ..keep.long]
dataSurv <- surv.split[, ..keep.surv]

Models for the longitudinal data

We shall compare two models for the biomarker data: a linear mixed model with random slopes, and a more general model linear Gaussian state space model with random slopes, autoregressive drift and diffusion (Wiener process).

## create list of models
models.list <- listmodels(biomarkers, timeinvar.long)[c(1, 5)]

Fitting the longitudinal model

Next we use ctsem to fit these models to the full dataset. The function jmseq::ctstanfit.fold() is a wrapper for ctsem::ctStanFit()


lmmonly.fit <- ctstanfit.fold(train.dataset=dataLong, ctmodel=models.list[[1]])
lmmdriftdiff.fit <- ctstanfit.fold(train.dataset=dataLong, ctmodel=models.list[[2]])

Comparison of longitudinal models

Comparing the fit of the full Gaussian state space model to the linear mixed model shows that the full model has higher log-likelihood and lower AIC. This is typical of clinical data with long follow-up times.

Generating forward updates of latent state values with the Kalman filter

From the fitted models we can generate forward updates of the predicted latent state trajectories for each individual. The ctsem::ctKalman() function has an option to plot these trajectories. Each time a new observation arrives, the predicted trajectories are updated: so the curve may jump to a new position.

Fitting the event model

To fit the event submodel, we need to generate imputed values of the latent state variables from the forward updates of the state probabilities computed by the Kalman filter.

The function jmseq::kalmanwide calls ctsem::ctKalman() to generate the imputed values, and reshapes the table of imputed values from long to wide format: one observation per person-time interval.

latent.updates <- kalmanwide(lmmdriftdiff.fit, subjects="all",
                             timestep="asdata",
                             timerange=c(0, maxtime))

Using the imputed latent state values in a Poisson regression model for event status

These latent state values at the start of the interval are plugged into a Poisson regression model for the event status at the end of each interval.

We have three options for fitting the Poisson regression model: the R function glm, ('stan=FALSE') to maximize the likelihood, using Stan to find the posterior mode ('stan=TRUE, vb=FALSE'), or a variational Bayes algorithm with Stan ('stan=TRUE, vb=TRUE') which generates samples from the approximate posterior. The Stan options are included as a first step in development of a one-step joint model, in which information from the event indicators feeds into the longitudinal submodel. The variational Bayes algorithm minimizes the Kullback-Leibler divergence between the true posterior and the approximating distribution my maximizing the "evidence lower bound" (ELBO).


## fit Poisson model to training datasets for each fold within each ctsem model
poisson.glm.beta <- fit.poissontsplit(latent.updates,
                                      dataSurv, 
                                      timeinvar.surv, biomarkers, splines=FALSE,
                                      stan=TRUE, vb=TRUE)

Cross-validation

Test-training split

For cross-validation of the prediction of events, we split the dataset into test and training folds. This tutorial uses 5 folds but the user can set nfolds to a lower value to save computation, or to a higher value for more accuracy. .
Each test fold contains all observations after the landmark time (5 years) on the individuals in that test fold. Each training fold contains all observations on those individuals who are not in the corresponding test fold. For those individuals who are in the corresponding test fold, the biomarker observations after the landmark time are set to missing.


ids.tosample <- unique(dataSurv[Time > landmark.time, id]) # & dataSurv$Time < max(dataSurv$Time))
ids.permuted <- base::sample(ids.tosample)
## test-train split for nfold cross-validation
folds <- cut(1:length(ids.permuted), breaks=nfolds, labels=FALSE)

## create list of ids in each testfolds 
ids.test <- vector("list", nfolds)
for(i in 1:nfolds) {
    ids.test[[i]] <- ids.permuted[folds==i]
}

## create training datasets
train.datasets <- vector("list", nfolds)
for(i in 1:nfolds) { # loop over folds
    train.datasets[[i]] <- vector("list", 3)
    names(train.datasets[[i]]) <- c("Surv", "Long", "ids.test")
    train.datasets[[i]]$Surv <- trainsplit.surv(ids.test=ids.test[[i]],
                                                dataSurv=dataSurv,
                                                landmark.time=landmark.time) 
    train.datasets[[i]]$Long <- trainsplit.long(ids.test[[i]],
                                                dataLong=dataLong, landmark.time,
                                                biomarkers)
    train.datasets[[i]]$ids.test <- ids.test[[i]]
}

Fitting the longitudinal models to training folds

Next we run ctsem to fit each of these models to each training fold. In each training fold, the forward updates continue to the end of the follow-up period including the person-time intervals in the test fold, where the biomarker observations have been dropped. The fitted models are saved in fitted.list. This step is parallelised over folds using parallel::parLapply.

## mclapply and foreach %dopar% fail -- child processes cannot open sockets

start <- Sys.time()

fitted.list <- vector("list", length(models.list))
names(fitted.list) <- names(models.list)
cat("Looping over ctsem models to fit to training datasets ...\n")
for(m in 1:length(models.list)) { # loop over models
    fitted.list[[m]] <- 
     #lapply(X=train.datasets, FUN=ctstanfit.fold, ctmodel=models.list[[m]]) 
     parallel::parLapply(cl, X=train.datasets, fun=ctstanfit.fold, ctmodel=models.list[[m]]) 
     gc()
    names(fitted.list[[m]]) <- paste0("fold", 1:nfolds)
}
parallel::stopCluster(cl)
cat("done, time ", difftime(Sys.time(), start, "mins"), "minutes\n")

Imputing latent state values over all person-time intervals

Next we generate imputed values of the latent state variables from the forward updates of the state probabilities computed by the Kalman filter. In each training fold, the forward updates continue to the end of the follow-up period including the person-time intervals in the test fold, where any biomarker observations have been set to missing. This step is parallelised over folds using parallel::mcmapply.

The list of fitted models is of length 2, and each element is a list of fitted models for each training fold.

## generate imputations from each training fold within each ctsem model
## we have to run the imputations up to maxtime for everyone, then censor later
start <- Sys.time()
cat("Looping over ctsem models to generate imputations from Kalman filter ...\n")
kalwide.list <- vector("list", length(models.list))
names(kalwide.list) <- names(models.list)
for(m in 1:length(models.list)) { # loop over models
    kalwide.list[[m]] <- parallel::mcmapply(FUN=kalmanwide,
                                fitted.list[[m]],
                                MoreArgs=list(timestep="asdata",
                                              timerange=c(0, maxtime)),
                                SIMPLIFY=FALSE)
}
cat("done, time ",  difftime(Sys.time(), start, "mins"), "minutes\n")
imputed.test <- kalwide.list[[2]][[1]][Subject %in% ids.test[[1]]]

## no need to censor imputations at censoring time for each individual. 
## test dataset: retain rows with ids in test fold and imputed values after landmark.time.
## train dataset: drop rows after landmark time for those in test fold
kalwide.test.list <- vector("list", length(models.list))
kalwide.train.list <- vector("list", length(models.list))
for(m in 1:length(models.list)) { # loop over models
    kalwide.test.list[[m]] <- mapply(FUN=function(k, ids.keep) 
        k[id %in% ids.keep & tstart > landmark.time], 
        kalwide.list[[m]],
        ids.test,
        SIMPLIFY=FALSE)

    kalwide.train.list[[m]] <- mapply(FUN=function(k, ids.drop)
        k[!(id %in% ids.drop & tstart > landmark.time)],
        kalwide.list[[m]],
        ids.test, 
        SIMPLIFY=FALSE)
}

Training a Poisson regression model for event status on imputed latent states

The latent state values at the start of the interval are plugged into a Poisson regression model for the event status at the end of each interval.

## fit Poisson model to training datasets for each fold within each ctsem model
coeffs.train <- vector("list", length(models.list))
for(m in 1:length(models.list)) { # loop over models
    coeffs.train[m] <- vector("list", length(nfolds))
    for(fold in 1:nfolds) { # loop over trainig folds
        coeffs.train[[m]][[fold]] <- fit.poissontsplit(kalwide.train.list[[m]][[fold]],
                                                   train.datasets[[fold]]$Surv,
                                                   timeinvar.surv, biomarkers, splines=FALSE,
                                                   stan=FALSE)
    }
}
names(coeffs.train) <- names(models.list)

Prediction of event status on test folds

The final step in the cross-validation is to use the Poisson regression model fitted to each training fold to generate predictions of event status on the test folds from the forward predictions of the latent states. Predictive performance is evaluated by concatenating observed and predicted event status over all test folds.

## generate predictions on test folds 
## elements of testdata.list are concatenations of all test folds 
testdata.list <- vector("list", length(models.list))
names(testdata.list) <- names(models.list)
for(m in 1:length(models.list)) { # loop over models
    ## vectorize over test olds: args before MoreArgs are lists of folds
    testdata.model.list <- mapply(FUN=predict.testdata,
                                  beta=coeffs.train[[m]],
                                  imputed.dt=kalwide.test.list[[m]],
                                  MoreArgs=list(dataSurv=dataSurv,
                                                timeinvar.surv=timeinvar.surv,
                                                biomarkers=biomarkers,
                                                landmark.time=landmark.time),
                                  SIMPLIFY=FALSE)
    testdata.list[[m]] <- data.table::rbindlist(testdata.model.list)
}

## generate summary table of predictive performance
predict.table <- NULL
for(i in 1:length(testdata.list)) { # loop over models
    stats <- tabulate.predictions(testdata.list[[i]])
    predict.table <- rbind(predict.table, stats)
}
predict.table <- data.table(Model=names(testdata.list), predict.table)
predict.table[, Predicted := as.numeric(Predicted)]
predict.table[, `Person-years` := as.numeric(`Person-years`)]
predict.table [, `Log score` := as.numeric(`Log score`)]
predict.table[, `C-statistic` := as.numeric(`C-statistic`)]

The cross-validated predictive performance of the event submodels is summarised in the table below. The evidence that one model is better than another can be assessed by the differences in test log-likelihood (known as the logarithmic score). This however doesn't tell us anything about the size of the difference in predictive performance between models. For this we can compare the C-statistic values, though it is more meaningful to compare the expected information for discrimination as described elsewhere. For the C-statistic to be valid, we ignore any censoring before the end of the last time interval in each individual and compare the observed event status with the probability of failure before the end of each person-time interval (obtained from the predictive model). See this page for an explanation of why this is correct. The total person-years of follow-up after 5 years (on which the test dataset is constructed) is thus greater than the true value of 623 person-years. This discrepancy can be made arbitrarily small by splitting the follow-up time into short person-time intervals, but this increases the computation time.
